About

Project Overview

Our goal was to demonstrate throwing and catching with robot arms (UR5e). For throwing, we developed both a human-defined throwing policy and one learned using trajectory optimization. To demonstrate catching, we implemented an arm that tracks the projectile motion of the block and moves to intercept it.

Methodology

Technical Approach

01 Throwing Strategy

Baseline: Naive Controller

Implemented a human-defined throwing motion using pre-programmed joint trajectories. This served as our baseline to compare against learned approaches.

Learned Controller: RL Optimization

Used Proximal Policy Optimization (PPO) to learn optimal throwing trajectories. The action space consists of cubic spline knots for active joints, allowing smooth trajectory generation.

02 Catching Strategy

Trajectory Prediction

The catcher arm tracks the thrown block's position and velocity in real-time, predicting its parabolic trajectory using physics-based motion models.

Motion Planning

Once the trajectory is predicted, inverse kinematics (IK) is used to compute joint angles that position the gripper at the interception point. The arm moves to the predicted catch location with smooth motion planning.

Key Components

  • Real-time position tracking using Webots Supervisor API
  • Forward kinematics (FK) for end-effector positioning
  • Inverse kinematics (IK) for joint angle computation
  • Gripper timing control for secure catching

03 Simulation Environment

Webots Platform

Developed in Webots R2023b with headless simulation support for fast training. The environment includes accurate physics simulation with realistic joint dynamics and gripper mechanics.

System Architecture

  • Gym Environment: Custom OpenAI Gym wrapper for RL training
  • Simulator: Headless Webots instance for fast episode evaluation
  • Controllers: Separate controllers for thrower, catcher, and evaluation
  • Data Pipeline: JSON-based communication between training loop and simulation
